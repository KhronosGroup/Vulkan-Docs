// Common Valid Usage
// Common to indirect drawing commands
// {addressRange} is the name of the address range parameter being validated
// {addressFlags} is the name of the address flags parameter being validated
  * [[VUID-{refpage}-{addressRange}-13097]]
    If the range specified by pname:{addressRange} is not bound completely
    to memory when accessed, pname:{addressFlags} must: not include
    ename:VK_ADDRESS_COMMAND_FULLY_BOUND_BIT_KHR
  * [[VUID-{refpage}-{addressRange}-13098]]
    If the buffer from which the range specified by pname:{addressRange} was
    created with ename:VK_BUFFER_CREATE_PROTECTED_BIT, and
    <<limits-protectedNoFault, pname:protectedNoFault>> is not supported,
    pname:{addressFlags} must: include
    ename:VK_ADDRESS_COMMAND_PROTECTED_BIT_KHR
  * [[VUID-{refpage}-{addressRange}-13099]]
    If the buffer from which the range specified by pname:{addressRange} was
    created without ename:VK_BUFFER_CREATE_PROTECTED_BIT, and
    <<limits-protectedNoFault, pname:protectedNoFault>> is not supported,
    pname:{addressFlags} must: not include
    ename:VK_ADDRESS_COMMAND_PROTECTED_BIT_KHR
  * [[VUID-{refpage}-{addressFlags}-13100]]
    pname:{addressFlags} must: not include both
    ename:VK_ADDRESS_COMMAND_STORAGE_BUFFER_USAGE_BIT_KHR and
    ename:VK_ADDRESS_COMMAND_UNKNOWN_STORAGE_BUFFER_USAGE_BIT_KHR
  * [[VUID-{refpage}-{addressRange}-13122]]
    If any buffer, which is bound to a range of slink:VkDeviceMemory that
    overlaps the range backing pname:{addressRange}, was created with
    ename:VK_BUFFER_USAGE_STORAGE_BUFFER_BIT, pname:{addressFlags} must:
    include ename:VK_ADDRESS_COMMAND_STORAGE_BUFFER_USAGE_BIT_KHR or
    ename:VK_ADDRESS_COMMAND_UNKNOWN_STORAGE_BUFFER_USAGE_BIT_KHR
  * [[VUID-{refpage}-{addressRange}-13123]]
    If any buffer, which is bound to a range of slink:VkDeviceMemory that
    overlaps the range backing pname:{addressRange}, was created without
    ename:VK_BUFFER_USAGE_STORAGE_BUFFER_BIT, pname:{addressFlags} must: not
    include ename:VK_ADDRESS_COMMAND_STORAGE_BUFFER_USAGE_BIT_KHR
ifdef::VK_EXT_transform_feedback[]
  * [[VUID-{refpage}-{addressFlags}-13101]]
    pname:{addressFlags} must: not include both
    ename:VK_ADDRESS_COMMAND_TRANSFORM_FEEDBACK_BUFFER_USAGE_BIT_KHR and
    ename:VK_ADDRESS_COMMAND_UNKNOWN_TRANSFORM_FEEDBACK_BUFFER_USAGE_BIT_KHR
  * [[VUID-{refpage}-{addressRange}-13124]]
    If any buffer, which is bound to a range of slink:VkDeviceMemory that
    overlaps the range backing pname:{addressRange}, was created with
    ename:VK_BUFFER_USAGE_TRANSFORM_FEEDBACK_BUFFER_BIT_EXT,
    pname:{addressFlags} must: include
    ename:VK_ADDRESS_COMMAND_TRANSFORM_FEEDBACK_BUFFER_USAGE_BIT_KHR or
    ename:VK_ADDRESS_COMMAND_UNKNOWN_TRANSFORM_FEEDBACK_BUFFER_USAGE_BIT_KHR
  * [[VUID-{refpage}-{addressRange}-13125]]
    If any buffer, which is bound to a range of slink:VkDeviceMemory that
    overlaps the range backing pname:{addressRange}, was created without
    ename:VK_BUFFER_USAGE_TRANSFORM_FEEDBACK_BUFFER_BIT_EXT,
    pname:{addressFlags} must: not include
    ename:VK_ADDRESS_COMMAND_TRANSFORM_FEEDBACK_BUFFER_USAGE_BIT_KHR
endif::VK_EXT_transform_feedback[]
